# ドキュメントインデックス

## 目的
カップル向けアプリのプロダクト方向性と検証計画に関する情報を集約する。

## ドキュメント一覧
- [`product-vision.md`](product-vision.md): 課題設定、価値提案、判断原則、成功指標。
- [`roadmap.md`](roadmap.md): アルファからベータまでの目標とマイルストーン。
- [`alpha-experiment-plan.md`](alpha-experiment-plan.md): 開発者ドッグフーディング用の仮説、指標、リサーチ計画。
- [`alpha-mvp-spec.md`](alpha-mvp-spec.md): 初期開発に向けた機能要件・非機能要件・未決事項。
- [`user-stories.md`](user-stories.md): 主要なふたりのジャーニーを整理したストーリー。
- [`data-model.md`](data-model.md): Supabase/Prisma向けのテーブル設計と拡張方針。
- [`screen-architecture.md`](screen-architecture.md): アルファMVPの画面構成とナビゲーション設計。
- [`screen-flow.md`](screen-flow.md): 認証状態とカップルスペース作成状態に基づいた画面遷移フロー。
- [`implementation-plan.md`](implementation-plan.md): App Router構成とフェーズ別の実装タスク整理。
- [`server-architecture.md`](server-architecture.md): サーバーサイドのレイヤードアーキテクチャ設計とリファクタリング指針。
- [`date-handling-guide.md`](date-handling-guide.md): Dateシリアライズ問題の背景と型定義・変換パターン。

## 運用のヒント
- 調査・検証のたびに更新し、最新の学びを反映する。
- Supabaseのスキーマやチケットなど関連資料ができ次第リンクを追加する。
- READMEに古くなった情報や要検討の論点があれば明記しておく。
