# アルファ実験計画

## 仮説
1. 家事リストに賞賛プロンプトを組み込むと、完了率と相互感謝が共に向上する。
2. 週2回の利用を想定した共有買い物リストは、買い忘れと重複購入を減らす。
3. 招待→参加を前提としたパートナー向けオンボーディングは、24時間以内の二人目参加率80%以上を達成できる。

## 成功指標
- **アクティベーション**: 招待から24時間以内にオンボーディングを完了したパートナーの割合。
- **エンゲージメント**: カップルあたり1週間で完了する共有タスク数の平均。
- **感謝の循環**: 完了から12時間以内に賞賛アクション（テキスト、スタンプ、リアクション）が送られた家事の割合。
- **リテンション指標**: カップル単位で週1回以上リスト更新を続けた連続週数。

## 計測チェックリスト
- 計測イベント: invite_sent, invite_accepted, chore_created, chore_completed, praise_sent, list_item_added/completed, notification_delivered, notification_opt_out。
- カップル単位のメタデータを保持し、賞賛エンゲージメントと継続率の相関を分析。
- クライアント側のパフォーマンスをログ化し、リアルタイム同期の体感レイテンシを400ms未満に保つ。

## 実験タイムライン
1. **スプリント0**: UXフロー確定、Supabase＋内製ダッシュボードで計測準備。
2. **スプリント1**: MVPをリリースし、開発者カップルで7日間ドッグフード、日次ふりかえり。
3. **スプリント2**: 信頼できる2組へ拡大、週中にフィードバック面談。
4. **スプリント3**: 通知頻度の調整、賞賛テンプレートの改善、NPSスタイルの簡易パルス調査。

## 学習アジェンダ
- どの口調・頻度の賞賛プロンプトが自然で、どこから不自然に感じるか。
- どちらかが“管理者役”に偏るのか。その場合の巻き込み策は何か。
- 招待／オンボーディングフローで最も摩擦が大きいポイントはどこか。
- 放置されがちなタスクの傾向（大きな家事 vs. 繰り返しの小さな家事など）。

## 定性リサーチ計画
- 実験前に各パートナーへ期待値ヒアリング。
- アプリ内の軽量プロンプト：「この家事で賞賛されてどう感じた？」（絵文字評価＋自由回答）。
- 週次のふりかえりで、感情面と習慣変化に焦点を当てたインタビュー。

## 終了条件
- 開発者カップル両名が、家事・買い物のメンタルロード軽減を実感している。
- 完了した家事の50%以上が“スパム感なく”賞賛されている（利用者自己申告含む）。
- 次フェーズに向けた最大の摩擦ポイント上位3件と改善案が明確になっている。
