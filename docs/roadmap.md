# プロダクトロードマップ

## フェーズ概要
- **アルファ（開発者＋そのパートナー）**: 共有タスクと賞賛ループの価値仮説を検証する。
- **ベータ（招待制の小規模コホート）**: 日常の習慣を拡張する補完機能を追加し、体験の広がりを試す。
- **第3フェーズ（TBD）**: 先行フェーズから得たインサイトを踏まえて広く展開する。

## アルファの目標（0〜2か月）
- 仮説の検証：共有タスク＋賞賛ループが実行率と満足度を高めるか。
- 二人向けオンボーディングの滑らかさと信頼性を確保する。
- 次フェーズの意思決定に使えるコア指標を計測する。

### アルファ機能スコープ
- カテゴリ管理とリアルタイム同期を備えた共有買い物リスト。
- 完了ログと賞賛ナッジを備えた共有家事ボード。
- パートナー招待／プロフィール編集を含むオンボーディング。
- 頻度・チャネルを調整できる通知システム。

### 主なマイルストーン
1. 1週目: UXフローと計測設計を確定。
2. 2〜3週目: MVP実装、開発者カップルでドッグフーディング。
3. 4週目: フィードバック反映、オンボーディングと賞賛体験を磨く。
4. 5〜6週目: 信頼できる2〜3組へ展開し、質的インタビューを実施。

## ベータの目標（3〜4か月）
- 日常家事以外のシナリオまで拡張しても感情的な差別化を維持できるかを検証。
- 計画・家計ワークフローを統合したいニーズの有無を評価。

### ベータ候補機能
- 旅行計画ボード：アイデア共有、行程、持ち物を共同管理。
- タスクや旅行と紐づく割り勘機能で公平な負担を可視化。
- 週次ストリークや賞賛傾向などのインサイトダッシュボード。

### 継続判断のゲート
- 週次アクティブカップルが5組以上、賞賛エンゲージメント率50%以上。
- 競合よりも精神的負担が軽くなったという定性的フィードバック。
- リアルタイム基盤と通知周りの技術的な拡張準備が整っていること。

## 第3フェーズの検討事項
- 収益モデル（サブスクリプション、プレミアム機能など）の決定。
- コア利用が確立した後にカレンダーやスマートスピーカー連携を検討。
- 賞賛テンプレートや関係性を深めるコンテンツ戦略の拡充。
