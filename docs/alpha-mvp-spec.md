# アルファMVP仕様

## スコープ概要
共有買い物リスト、家事ボード＋賞賛ループ、パートナーオンボーディングに集中し、旅行・割り勘機能はベータ以降に先送りする。

## 機能要件
### パートナーアカウント
- 二人のプロフィールが紐づくカップルスペースを作成できる。
- メールまたは共有コードで招待、招待リンクは72時間で失効。
- 表示名と希望する敬称・代名詞を編集可能。

### 共有買い物リスト
- アイテム作成時にカテゴリタグや自由記述ノートを追加できる。
- 両者の画面でリアルタイム同期、オフライン時は再接続までキューイング。
- 購入済みアイテムを非表示にするトグルとリスト全消去機能。

### 家事ボードと賞賛
- 締切日（任意）と繰り返し設定（例: 毎週・毎月）付きで家事を登録できる。
- 完了操作で賞賛プロンプトを表示（自動サジェスト文＋クイックリアクション）。
- 自由入力の賞賛メッセージと軽量リアクション（❤️ 👏 👍）を送信可能。

### 通知
- パートナーごとにプッシュ／メールを切り替え、既定は「デイリーダイジェスト」。
- ダイジェストモードでない場合、家事完了を即時通知。
- 繰り返し家事が24時間以上遅延した際のリマインダー。

## 非機能要件
- モバイルファーストのレスポンシブWeb体験。
- タスク更新の体感同期遅延を平均400ms未満に保つ。
- アクセシビリティ考慮（フォーカス、ハイコントラスト、スクリーンリーダーラベル）。

## 計測イベント
- `couple_created`, `partner_invited`, `partner_joined`
- `list_item_added`, `list_item_completed`
- `chore_created`, `chore_completed`, `praise_sent`, `praise_reacted`
- `notification_delivered`, `notification_snoozed`

## 未決事項
- 賞賛メッセージを、カップル外に共有できるようにするか完全非公開にするか。
- デイリーダイジェストと即時通知の既定頻度をどう設計すべきか。
- Supabaseで共有タスクをどうモデリングするか（単一テーブルか、リストと家事を分けるか）。

## 依存関係
- カップル単位でアクセス制御するSupabase認証とRLS設定。
- Supabaseチャネル等を用いたリアルタイム更新。
- OneSignalなどの通知配信サービスの導入可否を検証。
